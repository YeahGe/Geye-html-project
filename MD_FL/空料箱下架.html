【确认下架】

1.校验当前PDA系统参数的"工位编号"

       a.工位编号为'FLSJ01'或'FLSJ02'

             a1.通过当前校验

             a2.若工位编号 == FLSJ01 ：任务终点固定为 "FLCK-2-1-8-A"

             a3.若工位编号 == FLSJ02 ：任务终点固定为 "FLCK-2-15-8-A"

       b.否则

             b1.结束全部校验

             b2.Toast提示：请设置工位编号(FLSJ01、FLSJ02)

2.再次查询获取当前"立库闲置料箱"数量

         a.立库闲置料箱  = 0

               a1.Toast提示：暂无闲置料箱

               a2.结束执行

         b.立库闲置料箱 ＞0

               b1.通过当前校验

3.将当前"立库闲置料箱"数量与当前填写/选中数值比较

        a.数值 ＞立库闲置料箱

               a1.Toast提示：空闲料箱不足(闲置料箱数量：X)

               a2.清空当前自定义/数字按钮的选中

               a3.结束执行

        b.数值  ≤ 立库闲置料箱

             b1.查询"申请数量"个数的闲置料箱及对应的库位编号

                        b1_1.闲置料箱取值：闲置料箱定义中按'料箱二维码'排序前X个料箱

                        b1_2.库位取值："料箱号"于[基础数据系统>>仓库设置：辅料箱管理]中对应的"库位"

             b2.按获取的闲置料箱向立库下发多条指令[全部对接字段以文档为准，下述业务相关字段]

                        b2_1.指令条数：X条

                        b2_2.指令类型：OUT( 出库：立库库位→出库站台 )

                        b2_3.优先级：    0 (暂不做上下架穿插执行指令的逻辑)

                        b2_4.任务起点：由 3.b2_2 中获取的库位编码

                        b2_5.任务终点：由 1.a 中获取的"固定终点库位"

                        b2_6.托盘条码：由 3.b2_1 中获取的'料箱编码'

             b3.每单条下发指令应答

                        b3_1.Code=0 (成功)

                               ①Toast提示：请求成功(料箱编码)

                        b3_2.Code<0 (失败)

                               ①返回WCS提供的错误提示

             b4.每单条指令结果反馈

                        b4_1.resCode=0 (完成)

                               ①将对应"料箱二维码"于[基础数据系统>>仓库设置：辅料箱管理]的"库位"固定置成'下架暂存'

                               ②将对应"料箱二维码"于[基础数据系统>>仓库设置：辅料箱管理]的"当前存放区域"置成对应非绑定区域

                        b4_2.resCode=1 (异常) / 2 (取消)

                               ①暂不处理

               b5.刷新"立库闲置料箱"的取值

         a.手动刷新：点击【刷新】按钮进行获取

         b.自动刷新：每60s刷新下数值

2.数值描述：满足下述全部条件的[基础数据系统>>仓库设置：辅料箱管理]中料箱的数量

         b.当前存放批次==0

         c.不属于进行中(未反馈任务结果)的指令列表中的'托盘条码(palletCode)'
